ROADMAP DI IMPLEMENTAZIONE - ALLINEAMENTO APT PAPER (versione validata)
Data: 22 aprile 2026
Ambito: Action Genome, replica fedele del paper "Dynamic Scene Graph Generation via Anticipatory Pre-training"
Vincolo risorse: GPU locale di buona qualita', no HPC

Decisioni di scope (sessione di implementazione)
- Implementazione end-to-end (tutte le fasi, code-only, no training run).
- Protocollo unlabeled frames STRICT: indicizziamo anche i frame non-keyframe in action_genome/frames/
  e li facciamo passare dal detector per ottenere boxes/labels usati come storia {I_{t-1}}.
- Baseline STTran viene sostituita: sttran.py rimane come riferimento ma il pipeline primario
  passa per lib/apt_model.py; train.py/test.py diventano wrapper leggeri verso i nuovi script.
- Dimensioni del paper: object rep = 840 (M_o v (512) + phi(b) (128) + semantic (200)),
  relationship rep = 2192 (840 + 840 + 512).

Obiettivo finale
- Portare la repo da baseline STTran a implementazione allineata al paper APT,
  con pipeline a due stadi (pre-training anticipatorio + fine-tuning),
  componenti architetturali dedicate e risultati riproducibili.

FASE 2 - Task anticipatorio P(G_t | {I_{t-1}}) con frame unlabeled (STRICT)
Priorita': P0
Durata stimata: 3-4 giorni

Task
1. Indicizzare TUTTI i frame in action_genome/frames/<video_id>/, non solo quelli
   con key-frame annotation.
2. Per ciascun frame target t (key-frame etichettato), costruire la finestra di storia
   {I_{t-gamma}, ..., I_{t-1}} e {I_{t-lambda}, ..., I_{t-1}}.
3. Sampling 1 frame ogni 3 nella storia (paper).
4. Padding con copia del primo frame disponibile se la storia e' piu' corta della finestra.
5. Detector frozen su ogni frame della storia per ottenere box + category distribution.
6. Loss applicata SOLO sul frame target.

FASE 3 - Pair matching Eq. (10) e costruzione sequenze
Priorita': P0
Durata stimata: 2-3 giorni

Task
1. Implementare matching pair soggetto-oggetto cross-frame con score
   epsilon = min(IoU_subject, IoU_object), soglia > 0.8.
2. Per ogni pair (soggetto-oggetto) rilevato in I_{t-1}, tracciare a ritroso la sequenza
   gamma e la sequenza lambda tramite matching frame-by-frame.
3. Fallback/padding: se in un frame non c'e' pair corrispondente, copiare il match piu' vicino.
4. Restituire gli indici di pair per frame e la mappa sequenziale per short/long-term.

File da creare

File da modificare

Deliverable

FASE 4 - Progressive Temporal Encoder (short + long)
Priorita': P0
Durata stimata: 4-6 giorni

Task
1. Spatial Encoder (1 layer multi-head, head=8) su feature oggetti per frame
   f_{t,i} = [M_o v_{t,i}, phi(b_{t,i}), s_{t,i}] -> dim 840. Non serve positional encoding
   perche' phi(b) contiene gia' l'informazione spaziale.
2. Relationship feature e_{t,ij} = [fhat_{t,i}, fhat_{t,j}, M_u u_{t,ij}] -> dim 2192.
3. Semantic extractor c_{t',ij} (FC singolo) sul pair -> dim 2192 per allineamento token.
4. Short-term encoder (3 layer multi-head, head=8) con input
   X(0)_s,ij = [A_ij + Z_s ; C_ij] (concat lungo dim sequenza), A_ij di lunghezza gamma=4.
5. Long-term encoder (3 layer multi-head, head=8) con input
   X(0)_l,ij = {f_theta(U_ij), psi(Xhat_s,ij)} + Z_l dove:
   - U_ij ha lunghezza lambda=10
   - psi e' un MLP 3-FC ReLU applicato token-wise (non cambia lunghezza)
   - f_theta comprime la sequenza lambda in un token aggregato:
     f_theta(U_ij) = W_theta ( phi(u_{t-lambda,ij}) CONCAT ... CONCAT phi(u_{t-1,ij}) )
     dove phi e' un Conv1d(kernel=1) per reshape dimensionale e W_theta una Linear.
6. Frame encoding Z_s e Z_l: embedding learnable (no sinusoidal) sulle posizioni temporali.

File da creare
- lib/apt_temporal_encoder.py

File da modificare
- lib/apt_model.py (orchestrazione)

Deliverable
- Modulo temporale APT separato dal glocal transformer STTran.

Done quando
- Forward produce Xhat_s,ij e Xhat_l,ij con shape coerenti alle equazioni del paper.

FASE 6 - Global temporal encoder del fine-tuning
Priorita': P1
Durata stimata: 2-3 giorni

Task
1. Implementare encoder globale che CONDIVIDE i parametri con lo Short-Term encoder

3. Output: Xhat_g,ij -> xhat_g,ij = ultimo token -> Classifiers_fin.

FASE 7 - Protocollo training allineato al paper
Priorita': P1
Durata stimata: 1-2 giorni

Task
1. Pre-training: SGD momentum=0.9, lr=1e-3, decay=0.9 per epoca, batch size=16.
2. Fine-tuning: SGD momentum=0.9, lr=1e-5, decay=0.9 per epoca, batch size=16.
3. Iperparametri architetturali: gamma=4, lambda=10.
4. Sampling 1/3 nel pre-training sulle finestre di storia.
5. Loss: MultiLabelMarginLoss su TUTTE e tre le teste (attention, spatial, contacting).
   - Per frame target con set di relazioni Y+ (ground-truth) e Y- (negative) calcolo
     L = sum_{p in Y+} sum_{q in Y-} max(0, 1 - y^p + y^q).

File da modificare
- train_pretrain.py, train_finetune.py
- configs/apt_pretrain.yaml, configs/apt_finetune.yaml

Deliverable

Done quando

Rischi principali e mitigazioni
- OOM su GPU locale: gradient accumulation, riduzione batch, checkpointing attivazioni.
- Pair matching rumoroso: diagnostica IoU e debug su mini subset.
- Regressione baseline: mantenere tag baseline e smoke-test APT a ogni fase.
- Detector su unlabeled frames costoso: caching delle feature per frame su disco.

Criterio di successo finale
- Implementazione con componenti APT richieste dal paper,
- training a due stadi funzionante,
- valutazione completa Action Genome,
- confronto quantitativo tracciato e riproducibile.
