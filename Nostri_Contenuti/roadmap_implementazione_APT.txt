ROADMAP DI IMPLEMENTAZIONE - ALLINEAMENTO APT PAPER
Data: 22 aprile 2026
Ambito: Action Genome, replica fedele del paper "Dynamic Scene Graph Generation via Anticipatory Pre-training"
Vincolo risorse: GPU locale di buona qualita', no HPC

Obiettivo finale
- Portare la repo da baseline STTran a implementazione allineata al paper APT,
  con pipeline a due stadi (pre-training anticipatorio + fine-tuning),
  componenti architetturali dedicate e risultati riproducibili.

-------------------------------------------------------------------------------
FASE 0 - Baseline solida e criteri di replica
Priorita': P0
Durata stimata: 1-2 giorni

Task
1. Congelare baseline STTran esistente su Action Genome.
2. Definire target metriche da confrontare (PredCls, SGCls, SGGen; with/no constraint).
3. Stabilire formato logging unificato per stage e split.

File coinvolti
- train.py
- test.py
- lib/config.py
- Nostri_Contenuti/report.txt

Deliverable
- Esecuzione baseline documentata.
- Tabella di riferimento "prima vs dopo" per tutte le metriche.

Done quando
- Una run baseline completa e' ripetibile con stesso comando/config.

FASE 1 - Refactor pipeline in due stadi
Priorita': P0
Durata stimata: 2-3 giorni

Task
1. Separare training in due script:
   - stage 1: pre-training anticipatorio
   - stage 2: fine-tuning
2. Introdurre configurazioni distinte per i due stage.
3. Definire naming checkpoint separato (pre e fin).

File da creare

- train_finetune.py
- configs/apt_pretrain.yaml
- configs/apt_finetune.yaml

File da modificare
- lib/config.py (o sostituzione con parser config per stage)

Deliverable

- Checkpoint salvati separatamente per stage.

Done quando
- Stage 1 e stage 2 girano indipendentemente e in sequenza.

FASE 2 - Implementazione task anticipatorio P(G_t | {I_{t-1}})
Priorita': P0
Durata stimata: 3-4 giorni

Task
1. Costruire datapipe che fornisce frame corrente + sequenza frame precedenti.
2. Gestire frame unlabeled per pre-training secondo protocollo paper.
3. Applicare loss solo su frame target etichettati.

File da creare

File da modificare

Done quando

-------------------------------------------------------------------------------
FASE 3 - Pair matching Eq. (10) e costruzione sequenze
Priorita': P0
Durata stimata: 2-3 giorni

Task
1. Implementare matching pair soggetto-oggetto cross-frame con score:
   epsilon = min(IoU_subject, IoU_object), soglia > 0.8.
2. Creare sequenze short-term e long-term per pair.
3. Gestire fallback/padding in caso di pair assente nel frame.

File da creare
- lib/pair_matching.py

File da modificare
- train_pretrain.py
- train_finetune.py

Deliverable
- Utility testabile che genera sequenze pair-corrette per gamma/lambda.

Done quando
- Verifica su batch: pair tracking coerente su piu' frame.

-------------------------------------------------------------------------------
FASE 7 - Protocollo training allineato al paper
Priorita': P1
Durata stimata: 1-2 giorni

Task
1. Pre-training: SGD, lr iniziale 0.001, decay 0.9/epoca, batch size 16.
2. Fine-tuning: SGD, lr iniziale 1e-5, decay 0.9/epoca, batch size 16.
3. Impostare gamma=4 e lambda=10 in config.
4. Sampling 1 frame ogni 3 nel pre-training.

File da modificare
- train_pretrain.py
- train_finetune.py
- configs/apt_pretrain.yaml
- configs/apt_finetune.yaml

Deliverable
- Training regime coerente con paper.

Done quando
- Log iperparametri combaciano con specifiche paper.

Rischi principali e mitigazioni
- Rischio: OOM su GPU locale.
  Mitigazione: gradient accumulation, riduzione batch, checkpointing attivazioni.
- Rischio: pair matching rumoroso.
  Mitigazione: diagnostica IoU e debug su mini subset visualizzato.
- Rischio: regressione baseline.
  Mitigazione: mantenere branch baseline intatto e benchmark fisso a ogni fase.

Criterio di successo finale
- Implementazione con componenti APT richieste dal paper,
- training a due stadi funzionante,
- valutazione completa Action Genome,
- confronto quantitativo tracciato e riproducibile.
