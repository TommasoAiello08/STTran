VIDVRD → STTran (predcls-style) pipeline (this repo)
===================================================

This file documents the **tensor pipeline**: VIDVRD-style JSON → ``entry`` dict →
``STTranMultiHead`` / ``head="vidvrd"``. For **weights, env vars, Colab commands,
checkpoints, and git**, see **REQUIRED_ARTIFACTS.txt** (broader, inventory-style).

This file documents the concrete utilities added to connect a VIDVRD-style JSON
(trajectory-based) to the exact `entry` dict format consumed by `lib/sttran.py`.

High-level objective
--------------------
We want to train / run a relation predictor using:
  - GT boxes + GT object labels (predcls-style)
  - CNN/ROI visual features extracted from frames (Faster R-CNN backbone)
  - general directed pairs (P→O, O→O, P→P, O→P)
  - a VIDVRD predicate head (131 predicates; usually +1 background)

Key constraints from current STTran implementation
--------------------------------------------------
In `lib/sttran.py`, relation features are constructed as:
  - visual: 3×512 = 1536
  - semantic: 2×200 = 400 (GloVe-init embeddings)
  - total: 1936

The transformer is instantiated with embed_dim=1936, so any new predicate head
should take input dim 1936.

Data contracts (what STTran consumes)
------------------------------------
STTran consumes an `entry` dict with (minimum) keys:

  boxes:         FloatTensor [N,5]  rows = [frame_idx, x1,y1,x2,y2]
  labels:        LongTensor  [N]    GT object class ids
  pred_labels:   LongTensor  [N]    in predcls, identical to labels

  pair_idx:      LongTensor  [R,2]  each row = (subj_node_id, obj_node_id)
  im_idx:        FloatTensor [R]    frame index for each pair (0..T-1)

  features:      FloatTensor [N,2048]      ROI features per node
  union_feat:    FloatTensor [R,1024,7,7]  union ROIAlign per pair
  spatial_masks: FloatTensor [R,2,27,27]   rasterized pair boxes

Utilities added (files and responsibilities)
--------------------------------------------

1) vidvrd_predcls_input.py
--------------------------
Purpose: VIDVRD JSON → node/pair tensors + predicate targets.

Key functions:
  - parse_vidvrd_json_dict(vidvrd_json)
      Parses the sample schema:
        subject/objects (tid→category)
        trajectories[f] (list of {tid,bbox,generated})
        relation_instances (subject_tid, object_tid, predicate, begin_fid, end_fid)

  - build_sttran_nodes_from_vidvrd_frames(...)
      Produces:
        boxes  [N,5]
        labels [N]
        frame_offsets[f] (global node offset for each frame)
        tid_to_local_idx[f][tid] for fast relation resolution

  - build_vidvrd_pairs_from_relation_spans(...)
      Expands relation spans into per-frame positive pairs:
        (frame_idx, subj_global, obj_global, pred_id)
      Skips frames where a subject/object tid has no box.

  - add_sampled_negatives(...)
      Adds negatives per frame by sampling from all directed pairs not in the
      positive set, producing:
        pair_idx [R,2], im_idx [R], pred_target [R]
      Convention: pred_target=0 is background/no_relation.

  - build_vidvrd_predcls_entry(...)
      Convenience end-to-end helper:
        VIDVRD JSON + tensors + featurizer → (entry, pred_target, skipped)

2) vidvrd_predcls_featurizer.py
-------------------------------
Purpose: compute the visual tensors STTran needs from frames + boxes + pairs.

Class:
  - VidvrdPredclsFeaturizer(fasterRCNN, chunk_frames=10)
      Reuses the Faster R-CNN model (resnet101) already loaded by
      `lib/object_detector.detector`.

Forward:
  - inputs:
      im_data [T,3,H,W]
      im_info [T,3] or [1,3]  (scale is im_info[0,2])
      boxes [N,5]
      pair_idx [R,2]
      im_idx [R]
  - outputs:
      features [N,2048]
      union_feat [R,1024,7,7]
      spatial_masks [R,2,27,27]

This matches the predcls logic in `lib/object_detector.py`:
  - boxes are scaled by im_info[0,2] before ROIAlign
  - union_boxes use im_idx as ROIAlign batch index
  - spatial masks are rasterized in unscaled coordinate space (as in repo)

3) lib/sttran.py (small refactor)
---------------------------------
``STTran.forward`` signature (``head`` is **required** — no silent AG default):

  forward(entry, *, head: str, return_global_output: bool = False)

  - ``head="ag"`` — fills ``attention_distribution``, ``spatial_distribution``,
    ``contacting_distribution`` on ``entry`` (three AG readouts from one trunk pass).
  - ``head="vidvrd"`` — does **not** fill those three; exposes ``global_output`` when
    ``return_global_output=True`` for the external VIDVRD linear head.

If ``return_global_output=True``, returns ``(entry, global_output)`` where
``global_output`` is the per-pair transformer output (dim 1936) fed to ``vidvrd_head``.

**3 vs 1 heads:** each forward is either the **three** AG predicate linears **or** the
**single** VIDVRD head on the same trunk — not four parallel heads in one call. See
``sttran_multitask_heads.STTranMultiHead.forward``.

4) sttran_multitask_heads.py
----------------------------
Purpose: add a VIDVRD predicate head while keeping AG heads intact.

Class:
  - STTranMultiHead(sttran, num_vidvrd_predicates)
      Contains:
        vidvrd_head: Linear(1936, num_vidvrd_predicates)

Forward:
  - head="ag": behaves like stock STTran (fills attention/spatial/contact keys)
  - head="vidvrd": calls sttran(..., return_global_output=True) and then:
        vidvrd_logits = vidvrd_head(global_output)

Pre-flight validation (Colab / local)
---------------------------------------
``lib/vidvrd_pipeline_validate.py``:

  - ``validate_vidvrd_sample_pipeline(json_path=..., frames_dir=..., expected_hw=(480,854), ...)``
    Checks JSON vs disk **(H,W) == (height,width)**, runs the **same** preprocessing as
    the Action Genome loader (``prep_im_for_blob`` / ``im_list_to_blob``), then
    ``build_vidvrd_predcls_entry`` + tensor checks + optional ``STTranMultiHead`` forward.
  - ``validate_vidvrd_dataset_layout(dataset_root, video_id=..., frames_subdir="test_frames_480", json_subdir="test_480")``
    for the ``VIDVRD-DATASET_480/...`` layout.

CLI (from ``STTran`` repo root, so ``import lib`` resolves)::

  cd STTran
  python lib/vidvrd_pipeline_validate.py --dataset_root /path/VIDVRD-DATASET_480 --expected_hw 480,854

Or explicit paths::

  python lib/vidvrd_pipeline_validate.py --json /path/video.json --frames_dir /path/to/frames_dir

End-to-end demo
---------------
File: run_vidvrd_json_demo.py

What it demonstrates:
  - parse JSON
  - build boxes/labels/pairs/targets
  - compute features via Faster R-CNN backbone
  - run STTranMultiHead with head="vidvrd" to get logits

It supports a `--dummy_frames` mode (random frames) to validate plumbing without
downloading VIDVRD frames.

What you still need for real training
-------------------------------------
See **REQUIRED_ARTIFACTS.txt** (sections D, F, G) for the full checklist, env vars, and
Colab CLI. In short:

  - **Detector**: ``fasterRCNN/models/faster_rcnn_ag.pth`` (repo-internal path; gitignored).
  - **GloVe** under ``data/`` for STTran semantic inputs.
  - **AG init**: ``ckpts/sttran_predcls.tar`` (or compatible) loaded into ``STTran`` before
    wrapping ``STTranMultiHead``. Training entrypoint: ``colab/vidvrd_train_colab.py``.
  - **VIDVRD data**: frames + annotations; **fixed** vocab (e.g. 34 objects, 131 predicates
    + background), not ad-hoc ``sorted()`` over one JSON file.
  - **Training step**: ``lib/vidvrd_train_utils.train_step_vidvrd`` expects ``entry`` in
    **predcls** form (``ObjectClassifier`` copies ``labels`` → ``pred_labels``). Build
    ``entry`` like ``build_vidvrd_predcls_entry`` + featurizer, or match the tensor
    contract listed above.

``run_vidvrd_json_demo.py`` does **not** need ``AG_DATA_PATH``: label lists come from
``data/ag_bootstrap/`` via ``lib/ag_bootstrap.load_ag_label_bundle()``. Full Action Genome
dataloaders still require ``AG_DATA_PATH``.

Checkpoints (base vs VIDVRD fine-tune, Colab / Drive)
---------------------------------------------------
**Problem:** Fine-tuned tensors may be only ``vidvrd_head`` (small) while the trunk still
comes from ``sttran_predcls.tar``. Saving only the head is valid if reload always:
  1) builds ``STTran`` and loads the **unchanged** base ``.tar`` into ``.sttran``, then
  2) wraps ``STTranMultiHead`` and applies the head checkpoint.

**Implementation:** ``lib/vidvrd_checkpoint.py``

  - ``save_vidvrd_train_checkpoint(path, multi, mode="full"|"head_only", extra_meta=...)``
  - ``apply_vidvrd_checkpoint_to_model(multi, path)``
  - ``backup_file(src, dest_dir)`` — copy original ``ckpts/*.tar`` to Drive before training

Training skeleton (Colab)
-------------------------
Narrative summary; **full flag list and backup layout** are in **REQUIRED_ARTIFACTS.txt**
section G.

  - ``colab/vidvrd_train_colab.py`` — argparse entrypoint: copies ``--base_ckpt`` into
    ``<out_dir>/backups/`` (and tries ``data/backups/``), builds ``STTranMultiHead``,
    calls ``freeze_for_vidvrd_training`` (AG readout linears frozen by default), runs
    epochs. ``--synthetic`` = random tensors only (does **not** validate JSON/featurizer).
  - ``lib/vidvrd_train_utils.py`` — ``build_training_batch_from_vidvrd`` re-exports the
    same path as ``build_vidvrd_predcls_entry`` (connects training to this document);
    ``train_step_vidvrd``, freeze helpers, ``make_synthetic_vidvrd_entry``.

Recommended Colab habit:
  - Rely on ``vidvrd_train_colab.py`` backups under ``<out_dir>/backups/``, or manually
    ``backup_file("ckpts/sttran_predcls.tar", "/content/drive/MyDrive/sttran_backups/")``.
  - Each epoch: ``save_vidvrd_train_checkpoint`` with ``mode="full"`` (simplest resume)
    or ``mode="head_only"`` if you always pair with the same base ``.tar`` and record
    ``base_ckpt`` in ``extra_meta``.
